Bash笔记(二)

Date 五 03 一月 2014 By laoyi Category Linux. Tags Bash

本系列博文记录学习Shell终端Bash过程中的收获，主要为常用命令一览表及相关功能的记录。

变量

dollar sign

使用美元符标记变量, 变量大小写敏感。 常用的系统变量有：

$HOSTNAME, $USER, $UID, $HOME

$PS1

变量定义终端的提示符, 可通过设置该变量改变提示符的内容：

$PS1='\u@\h:\W$'

显示内容：用户@主机:当前目录$

$PATH

设定系统搜索执行命令的路径

使用自定义变量

直接对变量名赋值即可

MyVar=123

使用时加上美元符

echo $MyVar

使用变量时引号的差异

简单说来就是这样：

echo '$MyVar' // $MyVar

echo "$MyVar" // 123

set

查看当前所有变量

set

env

查看当前exported的变量

env

unset

取消设置某个变量

unset [variable]

export

对某个变量操作使其在子bash中亦可使用, 该操作引入变量对父bash无效

export [var_name]

变量使用的另外一种方式

有时候直接使用$var会有问题，这种情况下可以这样获取变量

tree="A big tree " #注意最后空格

echo ${tree}on the load. #连接处没有空格

Bash的options

设置在使用未绑定变量时显示错误

set -u

echo $a_variable_not_exist #显示错误信息，而不是显示空

恢复显示空内容

set +u

显示当前bash的所有options

echo $-

对于每个option的功能可查看man bash

嵌套Bash

例子

echo $(var=1; echo $var)

var1在内层的Bash中被声明并显示, 在外层的Bash中没有被绑定

另外一种使用方式

echo `var=1; echo $var`

使用反引号替代美元符和括号

一个好用的地方在于，可以临时改变工作目录

echo `cd /bin/; ls`

命令执行后当前外层的Bash工作目录没有变化

历史记录

bang bang

显示并执行最近的一次操作

！！

![string]

使用感叹号后接字符串，Bash自动搜索匹配最后一个符合条件的命令， 显示并执行

echo Hey

!ec #echo Hey

history

显示最近的number条命令

history [number]

![number]

通过history获取命令记录后，使用感叹号加数字， Bash显示并执行对应索引值的命令

![number]

Ctrl-r

最方便的历史命令恢复操作， 按Ctrl-r，逐个输入字符串，Bash自动搜索并显示符合条件的记录。 找到需要的命令按回车即可执行

历史文件

Bash记录当前环境的历史命令。 当退出时， Bash将命令操作存在历史文件中。 一般是在~/.bash_history中。

显示当前环境历史记录的最大长度

echo $HISTSIZE

显示历史记录文件的位置

echo $HISTFILE

显示历史记录文件的最大长度

echo $HISTFILESIZE

文件匹配

asterisk

匹配任意长度任意字符

ls *.txt #显示所有后缀为txt的文件

question mark

匹配一个任意字符

ls ?.txt #显示所有名称为一个字符后缀为txt的文件

square brackets

匹配一个中括号中的字符

ls [ABC].txt #A.txt/B.txt/C.txt

也可以设定范围，数字或字符

ls [0-9a-zA-Z].txt #匹配一个数字或大小写字符

可以用！表示没有

ls [!4].txt #去掉4.txt

echo的一个神奇作用

在空目录下

echo *

显示一个菊花

在有文件的目录下

echo *

显示所有文件（不包含隐藏文件）

在使用时如果要显示*可使用逃逸符或引号将菊花包括起来

重定向与管道

输出重定向

将标准输出(1)重定向到文件

[command...] > [filename] #同：[command...] 1> [filename]

上述操作会覆盖原有文件, 可设置Bash的set -C拒绝写入原有文件。

标准输出(1)追加至文件末尾

[command...] >> [filename] # [command...] 1>> [filename]

标准错误流(2)的重定向

[command...] 2> [destination]

标准错误流(2)重定向至标准输出流

[command...] 2>&1

输入重定向

标准输入流(0)的重定向

[command...] < [source] # [command] 0< [source]

输入流结束符的定义

[command...] << [sign]

例子

cat << Hey #读取输入直到遇到Hey结束

管道

管道将一个命令的标准输出作为下一个命令的标准输入

例子：显示/etc目录文件的末四个

ls /etc | tail -4

管道可嵌套

ls /etc | tail -4 | tac

关于管道没多少内容，不过发挥想象力可以利用管道和过滤器创造整个世界。
