Python 笔记（五）

Date Wed 25 September 2013 By laoyi Category Python. Tags Python

本系列博文记录学习Python过程的笔记和想法，简单而狗血，杂乱而无章，主要目的是方便来日查看。

使用的Python版本为2.7.3， 在GNU/Linux Ubuntu 12.04LTS操作系统下实现和测试

类的进阶玩法

构造函数：

构造函数的名称为 "__init__"，在对象被生成时自动调用：

class ComplexClass:
        def __init__(self, number):
                self.number = number

继承状态下的构造函数：

如果一个类继承了某基类，且没有重写构造函数，则基类构造函数被自动调用：

class ComplexClass:
        def __init__(self, number):
                self.number = number

class AnotherClass(ComplexClass):
        pass

cc = ComplexClass(100)
ac = AnotherClass()
#错误！构造函数需要两个参数
ac = AnotherClass(123)
#正确的做法

调用父类构造函数：

方法一：调用父类的非绑定构造函数：

class ComplexClass:
        def __init__(self, number):
                self.number = number

class AnotherClass(ComplexClass):
        def __init__(self):
                ComplexClass.__init__(self, 123)

cc = ComplexClass(number = 100)

ac = AnotherClass()

在子类的构造函数中调用了父类的非绑定版本构造函数，并将自身作为参数传入。

方法二：使用super函数

使用super函数需要将类定义为新式类，方法是加入'__metaclass__ = type'一行

super函数可自动调用父类的构造函数的绑定版本，具体方法参见示例：

__metaclass__ = type

class ComplexClass:
        def __init__(self, number):
                self.number = number

class AnotherClass(ComplexClass):
        def __init__(self):
                super(AnotherClass, self).__init__(123)

cc = ComplexClass(number = 100)

ac = AnotherClass()

经测试，在多继承的情况下super函数只会调用被继承的第一个基类的构造函数（继承列表最左），这个问题有待深究。

魔法方法：

基本的魔法方法有四个，__len__, __getitem__, __setitem__ 以及 __delitem__

分别用于获取对象长度，获取对应键的值，设置对应键的值，以及删除对应键的值

定义了上述的方法后分别可通过如下函数调用对应功能：

获取长度可使用len(object), 获取对应键的值可用object[index], 设置值可下标操作后直接赋值，删除可使用del object[index]

property：

property用于创建类的属性，适用于某些依赖其他几个数据成员的属性的创建，应用property可以简化存储过程。

假设有一个程序员类，程序员类有工作时长和单位工资，假设总工资为时长乘以单位工资，那么可以利用property定义一个总工资属性：

class Coder:
        def __init__(self, time, salary):
                self.time = time
                self.salary = salary
        def getTotalSalary(self):
                return self.time * self.salary
        total = property(getTotalSalary)

coder = Coder(100, 40)
print coder.total
#4000

property最多接受四个参数，分别为fget, fset, fdel, doc，设定对应的方法便可绑定功能，若不设定对应的方法则调用时会抛异常。（如上文例子，给total赋值会引发异常）

静态方法和类方法：

声明静态方法或者类方法的简单方式是使用装饰器，例子：

__metaclass__ = type

class ComplexClass:
        def __init__(self):
                self.number = 100
        @staticmethod
        def staticFuncA():
                print "static function called!"
        @classmethod
        def classFuncA(cls):
                print "class function called by ",cls

ComplexClass.staticFuncA()
#static function called!
ComplexClass.classFuncA()
#class function called by  <class '__main__.ComplexClass'>

cc = ComplexClass()
cc.staticFuncA()
#static function called!
cc.classFuncA()
#class function called by  <class '__main__.ComplexClass'>

类的静态方法基本上跟普通的函数没有区别。可通过类或者对象加以调用。调用静态方法不传入self参数，因此也就无法调用对象的数据成员。

类方法是比较特别的东西，传入的参数是类，而不是对象，这种在工厂模式的情况下非常有用，可以对调用方法的类进行区别。

__getattr__等方法：

四个方法： __getattr__ __setattr__ __getattribute__ 以及 __delattr__

分别在特性被访问且不存在、设置特性值、特性被访问以及特性被删除的时候被调用。

使用上述方法可拦截并自定义对特性的访问。

抄个例子就一目了然了：

#usr/bin/env python
# -*- coding:gb2312 -*-

class Rectangle:
        def __init__(self):
                self.width = 0
                self.height = 0
        def __setattr__(self, name, value):
                if name == 'size':
                        self.width, self.height = value
                else:
                        self.__dict__[name] = value
        def __getattr__(self, name):
                if name == 'size':
                        return self.width, self.height
                else:
                        raise AttributeError

rc = Rectangle()
print "size", rc.size
#(0,0)
rc.size = (100, 50)
print "size after setting:", rc.size
#(100, 50)
rc.width = 50
print "change width to 50:", rc.size
#(50,50)
print "width", rc.width
#50

注意的是这里定义了__getattr__方法，该方法是在特性被访问且不存在的时候被触发，因此对width和height特性的访问不会被拦截。

迭代器：

__iter__方法返回一个迭代器，具有next方法的对象称为迭代器。每次调用迭代器的next方法返回下一个值，当迭代结束时引发一个StopIteration异常。

比较生猛的做法是类本身实现了__iter__方法和next方法，这样就能通过返回其自身进行迭代操作。

迭代器的方便之处在于处理列表无法储存的无限长序列，通过迭代器可以逐个返回处理。

搞一个无限长的步进列表：

#usr/bin/env python
# -*- coding:gb2312 -*-

class StepList:
        def __init__(self, begin, step):
                self.begin = begin
                self.step = step
        def __iter__(self):
                return self
        def next(self):
                now = self.begin
                self.begin += self.step
                return now

sl = StepList(15, 20)
for elem in sl:
        if elem > 500:
                break
        print elem,
#15 35 55 75 95 115 135 155 175 195 215 235 255 275 295 315 335 355 375 395 415 435 455 475 495

如果没有指定终止条件（elem > 500)，那么循环会永远进行下去。

稍做修改我们就得到了自动终止的迭代器：

#usr/bin/env python
# -*- coding:gb2312 -*-

class StepList:
        def __init__(self, begin, step, end):
                self.begin = begin
                self.step = step
                self.end = end
        def __iter__(self):
                return self
        def next(self):
                now = self.begin
                if now > self.end:
                        raise StopIteration
                self.begin += self.step
                return now

sl = StepList(15, 20, 200)
for elem in sl:
        print elem,
#15 35 55 75 95 115 135 155 175 195
