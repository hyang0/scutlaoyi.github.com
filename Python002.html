Python 笔记（二）

Date Fri 13 September 2013 By laoyi Category Python. Tags Python

本系列博文记录学习Python过程的笔记和想法，简单而狗血，杂乱而无章，主要目的是方便来日查看。

使用的Python版本为2.7.3， 在GNU/Linux Ubuntu 12.04LTS操作系统下实现和测试

字典

字典类似C++中STL的map，可保存键/值映射关系。但Python的字典比C++的简便得多。

创建字典的几种方法

mapOne = {'name':'szy', 'age':85} #创建有俩映射的字典
mapTwo = {} #空字典
#dict函数创建字典
mapThree = dict(name='szy', age=85)

关于字典的基本函数

#键/值对的数量
len(mapOne)
#2
#取值
mapOne['name']
#'szy'
#赋值
mapOne['name'] = 'scutLaoYi'
mapOne['name']
#'scutLaoYi'
#删除映射
del mapOne['name']
#mapOne == {'age':85}
#检测是否存在映射关系
'age' in mapOne
#True
'name' in mapOne
#False
#新建映射只需要直接赋值，键不用预先存在
mapOne['name'] = 'szy'
#mapOne == {'name':'szy', 'age':85}

利用字典格式化字符串

这个功能非常好用，在大量使用到某个字典的时候可以不用每次去一个个赋值，只需要把字典作为参数传入即可 注意括号后有个s

formatStr = "name: %(name)s, age:%(age)s"
print formatStr % mapOne
#name: szy, age:85

字典其他常用函数

:clear

清空字典

mapThree.clear()
#mapThree == {}

:copy

浅复制，复制的仅仅是字典，当值为列表等对象时复制的是对象的引用

mapFour = mapOne.copy()

:get

获取值，若不存在则返回默认值

这个函数可以防止由于键/值对不存在导致的抛出异常 默认不存在返回None，可设定第二个参数为返回值

print mapThree.get('height')
#None
print mapThree.get("height", '149cm')
#149cm
print mapOne.get('name')
#szy

:items

获取所有键/值对的元组，以列表方式返回

it = mapOne.items()
print it
#[('age', 85), ('name', 'szy')]

:iteritems

返回所有键值对的迭代器

由于迭代器暂时没有开始研究，在此跳过

:keys

获取所有键的列表

print mapOne.keys()
#['age', 'name']

:values

获取所有值的列表

print mapOne.values()
#[85, 'szy']

:pop

返回键的值并删除键/值对

print mapOne.pop('name')
#szy
print mapOne
#{'age': 85}

:popitem

随机弹出并删除键/值对，用于遍历和清空字典且不知道键值对的情况

:setdefault(a, b)

键a不存在则设置为b，键a存在则返回a的当前值，均返回a的值

mapOne.setdefault('name', 'scutLaoYi')
#'scutLaoYi'
mapOne.setdefault('name', 'szy')
#'scutLaoYi'

赋值的其他形式

#调用函数返回的元组可直接给多个变量赋值
x, y, z = 1, 2, 3
x, y = y, x #值交换
#数量不一致时抛异常

布尔值

False: False, None, 0(所有数值类型), [], (), {}

True: 上述值外的一切均为True

条件判断

num = input("Enter a number:")
if num > 0:
        print "num > 0"
elif num < 0:
        if num < -10:
                print "num < -10"
        else:
                print "num < 0"
else:
        print "num == 0"

if numOne > 100 and numTwo < 50:
        print "nothing..."

if numOne < 100 or numTwo > 50:
        print "also nothing..."

新奇的条件判断

x is y #相同对象
x in y #y容器中包含x
0 < age < 100 #age在(0,100)范围内（比C++爽）

断言

当断言被触发的时候条件语句后面的字符串会被输出

num = input("number < 100")
assert num < 100, "Hey! num is not less than 100!"

简单的循环

while True:
        print "so many string ..."

range(0, 5)

        print i,

zip迭代

注意该函数终止于短序列的结束

names = ['a', 'b', 'c']
numbers = [1, 2, 3, 4, 5]
zip(names, numbers)
#[('a', 1), ('b', 2), ('c', 3)]
for name, number in zip(names, numbers):
        print name, number

获取索引值的循环

for index, string in enumerate(strings):
        #...

break, continue, else

else在循环正常退出(非break)后执行

num = 40
while num < 100:
        num = input("num < 100:")
        if num == 100:
                print "num == 100"
                break
else
        print "num >= 100"

列表推导式

Python一个牛逼的功能，可以大幅度拓展for循环的能力

基本语法： [(算术表达式用于获取结果) for arg1 in stage1 (for arg2 in stage2...) (if ...)]

[x * x for x in range(1, 10)]
#[1, 4, 9, 16, 25, 36, 49, 64, 81]
[x * y for x in range(1, 5) for y in range(1, 5, 2)]
#[1, 3, 2, 6, 3, 9, 4, 12]
[2 * x + y for x in range(1, 5) for y in range(0, 5, 2) if y % 2 == 0]
#[2, 4, 6, 4, 6, 8, 6, 8, 10, 8, 10, 12]

pass

无话可说的时候就用这个苟延残喘一下

while True:
        pass
# -_-\\\

exec

可将其后的字符串作为Python语句执行 这种做法在字符串为用户输入时非常危险 类似的还有os.system('...')，虽然我自己一直在用 ^_^

exec "print 'Hello!'"
#Hello!

默认情况下exec与外部代码共享命名空间 可以限定命名空间使其与外部隔离，方法是给个字典，键为变量名，值为变量值 限定命名空间可限制内层代码的作用范围，防止命名污染，有助于提高安全性

scope = {'x':"szy"}
exec "print x" in scope
#szy
